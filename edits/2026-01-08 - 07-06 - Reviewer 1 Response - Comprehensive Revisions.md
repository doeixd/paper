#### 1.1 "Justified Truth" - Deflationary Clarification (~100 words)

**Location:** Section 4.3, Level 2 definition (Line 474)

**Problem:** Reviewer couldn't understand the term even after previous clarification. Objected that "truth is not justified; beliefs or assertions are justified."

**User Guidance:** "Make it clear, you are justified to call your beliefs true if it is coherent with a given consensus network. It's deflationary."

**Solution:** Replaced existing terminological note with simpler, clearer explanation:

**Old:**
> "Justified Truth" refers not to truth itself being justified (truth is not the sort of thing that can be justified), but to a proposition's status as justified in the direction of truth—certified by a reliable network as the best available approximation to objective truth. Read it as "justified-as-true" rather than "true-and-justified."

**New:**
> "Justified Truth" is deflationary shorthand for a belief's epistemic status. To say a belief has "Justified Truth" status means: you are justified in calling that belief true because it coheres with a low-brittleness Consensus Network. This does not mean truth itself is justified (truth is not the kind of thing that can be justified). Rather, it means you are warranted in asserting the belief as true. The term names the highest practical epistemic status: justified assertion of truth within the best available network.

**Key improvements:**
- Leads with "deflationary shorthand"
- "You are justified in calling X true" (clearer grammar)
- Emphasizes assertion-warrant, not property of truth
- Removes confusing "direction of truth" metaphor

**Location:** Section 4.4.2, after eye evolution analogy (Line 601)

**Problem:** Reviewer worried: "If multiple local peaks exist, and we are 'trapped' on a local peak rather than the global maximum, doesn't this vindicate the isolation objection?"

**User Guidance:** "Spell out anti-relativism argument explicitly."

**Solution:** Added explicit three-part response addressing local peaks concern:

**Structure:**
1. **Constraint sharing:** All viable peaks share load-bearing structural elements required by constraints (eye evolution: 40+ solutions, all share light-focusing, photoreceptor array, signal transmission)
2. **Objective ranking:** Peaks ranked by measurable brittleness ($P(t)$, $C(t)$, $M(t)$) — not epistemically symmetric
3. **Attractor basin depth:** No single global maximum; basins vary in depth. Low-brittleness systems in deep basins earn higher warrant than high-brittleness systems in shallow basins

**Conclusion:** Constrained pluralism, not relativism. Where constraints permit multiple stable solutions, accept genuine diversity. But diversity is bounded by shared structural requirements and ranked by objective performance.

**Rationale:**
- Directly addresses reviewer's specific worry
- Shows pluralism ≠ relativism via shared structure + objective ranking
- Integrates with attractor basin framework (Phase 1.2)
- Makes implicit argument explicit with clear logical structure

#### 2.2 Lakatos - Explicit Causal Engine Mechanism (~300 words)

**Location:** End of Section 6.4.1 (after Line 960)

**Problem:** Reviewer asked: "While Lakatos provides a brilliant historical description of a degenerating programme... our framework aims to provide the underlying causal engine for this degeneration. Where is that causal engine described though?"

**User Guidance:** "Strengthen Lakatos distinction."

**Solution:** Added explicit 8-stage causal mechanism showing how brittleness generates Lakatosian degeneration:

**The Causal Engine:**
1. Initial misalignment (hard core + protective belt structural features misalign with constraints)
2. Pragmatic pushback (manifests as failed predictions, anomalies, limited exportability)
3. Defensive response (add auxiliary hypotheses to protective belt rather than revise hard core)
4. Rising maintenance burden (each patch carries ongoing costs: tuning, tracking, consistency)
5. Cost accumulation ($M(t)$ rises, $P(t)$ rises, $C(t)$ may rise)
6. Diminishing returns (new predictions harder; each phenomenon requires new auxiliary structure)
7. Unsustainability (operational costs exceed benefits; practitioners migrate to alternatives)
8. Abandonment (programme enters Negative Canon)

**Example:** Phlogiston chemistry followed exact trajectory—initial elegance → negative weight patch → proliferating parameters → defensive publications → oxygen alternative → abandonment.

**Key distinction:** Lakatos retrospectively identifies stages 3-7 after the fact. Brittleness metrics track stages 4-6 in real-time (forward-looking diagnosis).

**Rationale:**
- Makes explicit what was previously claimed
- Shows causal process: misalignment → pushback → patching → costs → abandonment
- Demonstrates framework adds substantive mechanistic insight beyond Lakatos
- Concrete example (phlogiston) illustrates each stage

**Rationale:**
